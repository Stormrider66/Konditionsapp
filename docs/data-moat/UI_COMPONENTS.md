# Data Moat UI Components Specification

## Overview

This document specifies the UI components required to capture data for the Data Moat system. Components are designed to integrate seamlessly into existing workflows with minimal friction.

**Design Principles**:
1. **Minimal friction** - Capture data without interrupting workflow
2. **Contextual** - Appear at the right moment with relevant options
3. **Optional depth** - Quick capture with option for detailed input
4. **Feedback** - Show value back to users (accuracy, insights)

---

```typescript
  reasonCategories: DecisionReason[];  // Allow multiple
  reasonNotes?: string;
  coachConfidence: 'low' | 'medium' | 'high';

```

### Behavior

1. **Auto-trigger**: Shows when coach saves modified AI suggestion
2. **Can be skipped**: "Skip" dismisses without recording (but tracks skip rate)
3. **Quick path**: Single click on reason + Save = <5 seconds
4. **Remembers**: Last used reasons shown first
5. **Context auto-fill**: Pulls athlete's current metrics

### Integration Points

```typescript
// In WorkoutEditor.tsx
const handleSaveWorkout = async (workout) => {
  if (wasAIGenerated && hasModifications) {
    setShowDecisionModal(true);
    setPendingWorkout(workout);
  } else {
    await saveWorkout(workout);
  }
};

const handleDecisionSave = async (decision) => {
  await Promise.all([
    saveWorkout(pendingWorkout),
    saveCoachDecision({
      ...decision,
      aiSuggestionData: originalAIWorkout,
      modificationData: pendingWorkout,
      athleteId,
      workoutId: pendingWorkout.id,
    }),
  ]);
  setShowDecisionModal(false);
};
```

- Use shadcn/ui components as base
- Follow Tailwind CSS patterns in codebase
- Match existing color scheme and spacing
- Responsive design (mobile-first)
- Accessible (WCAG 2.1 AA)
